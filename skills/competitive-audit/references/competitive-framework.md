# Competitive Analysis Framework

## Step 1: Identify competitors
- **Direct**: Same problem, same audience
- **Indirect**: Same problem, different audience (or vice versa)
- **Aspirational**: Different space, gold-standard experience

## Step 2: Feature matrix
| Feature | Us | Competitor A | Competitor B | Notes |
|---------|-----|-------------|-------------|-------|
| [feature] | [status] | [status] | [status] | [insight] |

Status: full | partial | missing | planned

## Step 3: UX pattern analysis
For each key flow, document:
- How many steps to complete the task
- What information is required upfront vs. progressive
- What feedback is given at each step
- How errors are handled

## Step 4: Scoring
| Dimension | Weight | Us | Comp A | Comp B |
|-----------|--------|-----|--------|--------|
| Core functionality | 30% | | | |
| UX quality | 25% | | | |
| Performance | 15% | | | |
| Accessibility | 15% | | | |
| Ecosystem/integration | 15% | | | |

## Step 5: Gap analysis
- Where are ALL competitors converging? (table stakes)
- Where is NO competitor strong? (opportunity)
- Where are we uniquely positioned? (defend)
