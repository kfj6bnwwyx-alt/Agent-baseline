# Usability Testing

Adapted from designer-skills (MC Dean): usability-test-plan.

## Test plan structure

```markdown
# Usability test: [feature/flow]
## Objective: What are we trying to learn?
## Participants: [N] users matching [persona]
## Tasks:
1. [Task]: Find [X] and [do Y]
   Success criteria: [measurable]
   Time limit: [minutes]
2. [Task]: ...
## Metrics:
- Task completion rate (target: >80%)
- Time on task (target: <[N] minutes)
- Error rate
- Satisfaction (SUS or single ease question)
## Facilitation notes:
- Don't lead. "Where would you look?" not "Click the menu"
- Note hesitation, backtracking, confusion
- Ask "what are you thinking?" when they pause
```

## Reporting format
- Top 5 findings ranked by severity × frequency
- Video clips of key moments (if recorded)
- Heatmap of where users got stuck
- Recommendations tied to specific findings
