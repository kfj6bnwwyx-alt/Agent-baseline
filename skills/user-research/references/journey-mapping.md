# Journey Mapping

Adapted from designer-skills (MC Dean): journey-map.

## Journey map structure

| Stage | [Stage 1] | [Stage 2] | [Stage 3] | [Stage 4] |
|-------|-----------|-----------|-----------|-----------|
| **Doing** | Actions taken | | | |
| **Thinking** | Questions, concerns | | | |
| **Feeling** | Emotional state (↑ ↓ →) | | | |
| **Touchpoints** | Channels, tools, people | | | |
| **Pain points** | Friction, confusion, failure | | | |
| **Opportunities** | Where we can improve | | | |

## Guidelines
- Map the CURRENT journey first, then the ideal
- One journey per task or goal (not the entire product)
- Include moments BEFORE and AFTER your product
- Mark the "moment of truth" — the make-or-break point
- Annotate with evidence source (interview #, analytics, support ticket)
