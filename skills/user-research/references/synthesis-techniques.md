# Research Synthesis Techniques

Adapted from designer-skills (MC Dean): affinity-diagram, jobs-to-be-done.

## Affinity diagramming
1. Write every observation on a sticky (one insight per sticky)
2. Group silently — let clusters emerge, don't force categories
3. Name each cluster after grouping (not before)
4. Look for clusters that cut across multiple participants
5. Prioritize clusters by frequency and severity

## Jobs to Be Done (JTBD)

**Job statement format:**
When [situation], I want to [motivation], so I can [outcome].

**Three dimensions:**
- Functional: What task are they completing?
- Emotional: How do they want to feel?
- Social: How do they want to be perceived?

## Insight format

```markdown
## Insight: [title]
**Evidence:** [N] participants mentioned this
**Pattern:** [what we observed repeatedly]
**So what:** [why this matters for the product]
**Now what:** [recommended action]
```
