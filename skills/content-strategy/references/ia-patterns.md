# Information Architecture Patterns

## Navigation patterns

**Hub and spoke**: Central hub with branches. Good for distinct feature areas.
Example: Settings page with categories.

**Nested hierarchy**: Tree structure. Good for large content sets.
Example: Documentation sites, file systems.

**Flat**: All items at same level. Good for small sets (<10 items).
Example: App tabs, tool palette.

**Faceted**: Multiple independent dimensions. Good for search/filter.
Example: E-commerce (color + size + price + brand).

## Content organization models
- **Alphabetical**: When users know the exact name
- **Chronological**: Events, history, feeds
- **Geographical**: Location-based content
- **Task-based**: Organized by what users want to DO
- **Audience-based**: Different paths per user type

## Labeling principles
- Use user language (from card sorts and search logs)
- Be specific ("Billing" not "Account")
- Be consistent (if one tab is a verb, all are verbs)
- Front-load the distinguishing word ("Monthly report" not "Report, monthly")
- Test with tree testing (can users find things without the UI?)
