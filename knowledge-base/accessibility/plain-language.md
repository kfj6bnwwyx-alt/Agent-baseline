# Plain Language Guidelines

Adapted from inclusive-design-skills (MC Dean).

## Rules
- **Reading level**: Target 8th grade (age 13-14). Use Hemingway Editor to check.
- **Sentence length**: Average 15-20 words. Max 25.
- **Paragraphs**: 3-5 sentences max.
- **Active voice**: "Click the button" not "The button should be clicked"
- **Common words**: "use" not "utilize", "start" not "initiate", "help" not "facilitate"
- **One idea per sentence**: If you used "and" or "but", consider splitting.

## UI text specifically
- **Labels**: 1-3 words. Noun or verb phrase. Not sentences.
- **Buttons**: Start with a verb. "Save changes" not "OK". "Delete account" not "Continue".
- **Errors**: Say what happened + what to do. "Password too short. Use at least 8 characters."
- **Empty states**: Say what will appear here + how to get started.
- **Confirmations**: Name the action specifically. "Delete this project?" not "Are you sure?"

## Avoid
- Jargon without explanation
- Double negatives ("not uncommon")
- Idioms (they don't translate well)
- Abbreviations without first spelling out
