# Design Critique Framework

## Format
- **Presenter**: 5 min max. Show the work, state the goal, name what
  feedback you want ("Is the flow clear?" not "What do you think?").
- **Clarifying questions**: 2 min. Questions only, no opinions yet.
- **Feedback round**: Each person states one observation + one suggestion.
  Format: "I notice [observation]. Have you considered [suggestion]?"
- **Discussion**: Open. Presenter decides what to act on.

## Critique principles
- Critique the work, not the person
- Reference the contract or pattern when applicable
- "This doesn't match the contract" is actionable. "I don't like it" is not.
- If there's no contract yet, critique helps shape what the contract should say
- Accessibility concerns are not optional feedback — they're requirements

## Anti-patterns
- Design by committee (everyone's opinion has equal weight)
- Critique without context (not knowing the constraints)
- Solutioning in critique (save solutions for follow-up)
