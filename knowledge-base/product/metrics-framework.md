# Metrics Framework

## Product metrics (per feature)
| Category | Metric | How to measure |
|----------|--------|---------------|
| Adoption | Active users | Analytics |
| Engagement | Task completion rate | Analytics + usability tests |
| Quality | Error rate | Error tracking |
| Satisfaction | SUS score or single ease question | Survey |
| Performance | Page load time | Lighthouse |

## Design system metrics
| Metric | Target | How to measure |
|--------|--------|---------------|
| AI compliance rate | >90% | % generated code using system components |
| Token coverage | >95% | % UI values mapped to tokens |
| Contract coverage | >80% | % components with YAML contracts |
| Drift velocity | <5/sprint | New off-system references per sprint |
| Context file freshness | <7 days | Days since last L3 regeneration |
| A11y compliance | 100% AA | Automated + manual audit |

## How to set targets
1. Measure current baseline (before intervention)
2. Set target based on industry benchmark or team aspiration
3. Define measurement method and cadence
4. Track trend, not just snapshot
