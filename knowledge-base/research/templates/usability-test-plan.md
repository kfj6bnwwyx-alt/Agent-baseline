# Usability Test Plan Template

```markdown
# Usability test: [feature/flow name]

## Objective
What are we trying to learn? (Not "is it usable" — be specific)

## Participants
- Number: [5-8 recommended]
- Criteria: [matching which persona, with what experience level]
- Recruitment method: [how and where]
- Compensation: [amount and form]

## Tasks

### Task 1: [name]
**Scenario:** "[Realistic context setting]"
**Instruction:** "[What to do, without revealing HOW]"
**Success criteria:** [Binary — did they complete it?]
**Time limit:** [minutes]
**What to observe:** [specific behaviors to note]

### Task 2: [name]
...

## Metrics
| Metric | Method | Target |
|--------|--------|--------|
| Task completion rate | Binary per task | >80% |
| Time on task | Stopwatch | <[N] minutes |
| Error rate | Count per task | <[N] errors |
| Satisfaction | Single ease question (1-7) | >5 |

## Facilitation notes
- Don't lead: "Where would you look?" not "Click the menu"
- Note hesitation, backtracking, verbal confusion
- Ask "What are you thinking?" when they pause >5 seconds
- Don't help unless they're completely stuck (note it as a failure)

## Reporting
- Top 5 findings ranked by severity x frequency
- Video clips of key moments (with permission)
- Recommendations tied to specific observations
- Design system impact: any contract changes needed?
```
